01프로젝트 개요 & 목표

비전

전 세계 사용자가 어떤 언어, 어떤 디바이스에서든 가장 편리하고 아름답게 사용할 수 있는 올인원 온라인 도구 플랫폼. 광고 수익 기반의 무료 서비스로 운영하며, 압도적 UX와 완벽한 SEO로 오가닉 트래픽을 극대화한다.

핵심 목표

10+
서비스 도메인
200+
개별 도구
30+
지원 언어
100%
클라이언트사이드 처리

핵심 원칙

  • Privacy First — 파일 업로드 시 서버 전송 없이 100% 브라우저 내 처리 (WASM 활용)
  • Zero Sign-up — 회원가입 없이 모든 기능 무료 사용
  • Instant Load — 각 도구 페이지 LCP < 1.5초 목표
  • Mobile First — 모바일 사용자 60%+ 대비 설계
  • Global First — 30개 이상 언어 지원, 자동 감지

수익 모델

  • Google AdSense 디스플레이 광고 (메인)
  • 프리미엄 기능 잠금 해제 (선택적, 향후)
  • 제휴 마케팅 (관련 소프트웨어 추천)

  • 통합 플랫폼: 여러 사이트를 전전하지 않고 하나의 브랜드 아래에서 모든 도구를 사용

  • 완벽한 글로벌화: 30개+ 언어, 자동 감지, 각 지역 SEO 완벽 대응
  • 프라이버시: 100% 클라이언트 사이드 처리로 파일 업로드 불안 해소
  • 제로 제한: 회원가입 없음, 파일 크기 제한 없음(브라우저 메모리 한계 내), 워터마크 없음
  • 모바일 최적화: 터치 친화적 인터페이스, 반응형 100%
  • PWA + 오프라인: 앱처럼 설치, 인터넷 없이도 사용 가능

목표

일회성 방문이 아닌 반복 방문을 유도. 도구 플랫폼 특성상 "필요할 때 떠올리는 브랜드"가 되는 것이 핵심.

리텐션 전략

즐겨찾기 & 최근 사용
자주 사용하는 도구를 즐겨찾기하고, 최근 사용 도구에 빠르게 접근. 메인 페이지에서 개인화된 도구 목록 표시.
LocalStorage개인화
작업 히스토리
이전 작업 이력 확인 가능. "3일 전에 변환한 PDF를 다시 다운로드" 같은 편의 기능. 파일 자체는 저장하지 않고 메타데이터만.
IndexedDB편의성
PWA 설치 유도
3회 방문 시 부드러운 설치 프롬프트. 설치하면 앱 아이콘으로 접근성 극대화. 데스크톱 + 모바일 모두.
PWA리텐션

게이미피케이션데이터

추천 엔진도구 연계
키보드 단축키
파워 유저를 위한 키보드 단축키 지원. Ctrl+K로 도구 검색, 단축키 치트시트 제공.
파워유저편의성
배치 처리
여러 파일을 한번에 드래그 앤 드롭하여 일괄 처리. 프로그레스 바로 진행 상태 표시.
생산성효율

  • Auto Ads: AdSense 자동 광고를 기본으로, 수동 배치와 혼합
  • 광고 밀도: 페이지 콘텐츠 대비 30% 이하 유지 (AdSense 정책 + UX)

16개발 로드맵 & 일정

Phase 0: 기반 구축

Week 1~2

모노레포 세팅 (Turborepo + pnpm), 공유 패키지 초기화 (ui, design-tokens, i18n, seo, analytics, ads, storage, config), CI/CD 파이프라인 구축, Vercel 프로젝트 세팅, Cloudflare DNS 설정

Phase 1: 디자인 시스템 & 공통 컴포넌트

Week 3~4

Design Token 정의, 공유 UI 컴포넌트 개발 (Button, Input, FileUploader, ToolCard, Header, Footer, AdBanner, ThemeToggle, LanguageSwitcher, CommandPalette 등), 다크/라이트 테마, 모션 시스템 구축

Phase 2: 랜딩 + 첫 번째 앱 (PDF)

Week 5~8

메인 랜딩 페이지 개발, PDF 앱 전체 기능 구현 (18개 도구), i18n 5개 언어 적용, SEO 최적화, AdSense 연동, PWA 설정. PDF는 트래픽이 가장 높은 카테고리이므로 첫 번째로 개발.

Phase 3: Image + Video 앱

Week 9~13

Image Tools 전체 기능 (WASM Sharp 통합), Video Tools 전체 기능 (FFmpeg.wasm 통합). 두 앱 모두 파일 처리가 핵심이므로 WASM 인프라 공유.

Phase 4: Calculator + Text + Dev 앱

Week 14~18

계산기, 텍스트 도구, 개발자 도구 앱 개발. 파일 처리 없이 순수 로직 위주이므로 상대적으로 빠른 개발. SEO 키워드 볼륨이 높은 계산기 도구 우선.

Phase 5: Audio + QR + Convert + Color + Write + File 앱

Week 19~26

나머지 앱들 순차 개발. Audio (FFmpeg.wasm 공유), QR (가벼운 라이브러리), Convert (LibreOffice WASM 또는 서버 사이드), Color, Write, File.

Phase 6: i18n 확장 + SEO 강화

Week 27~30

30개+ 언어 번역 완료. SEO 콘텐츠 (FAQ, How-to, 블로그) 작성. 구조화 데이터 검증. Google Search Console 최적화.

Phase 7: 최적화 & 런칭

Week 31~34

성능 최적화 (Core Web Vitals), 크로스 브라우저 테스트, E2E 테스트 보강, 에러 모니터링 (Sentry), ProductHunt 런칭, 소셜 마케팅.

Phase 8: 지속적 개선 (런칭 후)

Ongoing

사용자 피드백 반영, 새 도구 추가, 광고 A/B 테스트, SEO 성과 분석 및 개선, 번역 품질 개선, 성능 모니터링.
